            Fetch TPL reads a stored template from R307 library slot N
            (first byte = slot id, rest is the compressed minutiae file ~256–768 B).
            Live CHAR captures a fingerprint NOW, runs Img2Tz, returns
            the live characteristic — same opaque proprietary blob, no slot prefix.
            Heatmap colours each byte (0 = empty, 255 = signal) so you can eyeball
            template density vs. zero padding.

          How is the verdict decided? The R307's onboard matcher
          compares the two characteristic files in CharBuffer1 and 2;
          cc=0x00 means the chip accepted them as the same
          finger. The score is the chip's own confidence metric (16-bit
          field, theoretical max 65535) — it scales with how many
          minutiae aligned. The chip does not document its threshold or
          practical range, so the bands below are observation-based
          buckets, not a calibrated verdict line.
